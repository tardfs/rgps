\section{Измерения с помощью одного приемника}
\subsection{Анализ файла RS-twin-15mm-01}

\begin{verbatim}
<RAW MEASURMENTS COLLECTED:> 228
<UBX_ECEF:> 1308
<UBX GEODETIC:> 1308
\end{verbatim}

Условные названия алгоритмов: 
\begin{itemize}
\item ubx - это измерения полученные с помощью приемника (собственный алгортм приемника) - на карте изображен зелеными метками
\item gnss - стандартный LS метод решения навигационной задачи - на карте голубой
\item easy - метод решения навигационной задачи при котором на каждой итерации уточняется время и пересчитывается позиция всех спутников - на карте изображен красным цветом 
\end{itemize}


\begin{figure}[h]
%\centering
\includegraphics{map1.png}
\caption{Измерения ubx(зеленый), easy(красный) и gnss(голубой), измерения приведены не полностью!}
\end{figure}

Алгоритм gnss имеет смещение, природа этого смещения пока мне не ясна - разбираться сейчас в этом я не вижу особого смысла. 
Отличие ubx от easy я на данный момент вижу в том что:

\begin {enumerate}
\item ubx использует больше спутников за счет GLONASS
\item ubx наверняка выбирает лучшие спутники учитывая их углы азимута и элевации - у меня в тукущей реализации этого нет.
\item ubx использует фильтрацию, которая при неподвижном приемнике очень сильно подавляет разброс. Остается неясным,
почему easy координаты после осреднения не стремятся к ubx координатам. В случае полнотью неподвижного приемника идеальный
фильтр - это осреднение.
\end{enumerate}
